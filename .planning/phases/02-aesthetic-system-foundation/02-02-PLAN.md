WallLabel (src/components/aesthetic/WallLabel.tsx):
```typescript
interface WallLabelProps {
  title?: string
  artist?: string
  medium?: string
  year?: string | number
  dimensions?: string
  children?: ReactNode  // Free-form inline variant
  size?: 'sm' | 'md' | 'lg'
  className?: string
}
export function WallLabel(props: WallLabelProps): JSX.Element
```
Behavior: if `children` is provided, render an inline `<span>` with `font-label uppercase tracking-[0.18em] text-ink` + size class. Otherwise render a `<div>` block with artist (uppercase tracked bold), title (italic, year appended after comma), medium (text-ink-soft), dimensions (text-xs text-ink-soft).

Receipt (src/components/aesthetic/Receipt.tsx):
```typescript
interface ReceiptProps {
  header?: string
  subheader?: string
  children: ReactNode
  stamped?: boolean
  className?: string
}
export function Receipt(props: ReceiptProps): JSX.Element

export function ReceiptRow({ label, value }: { label: string; value: string | number }): JSX.Element
```
Behavior: framer-motion.div with `initial={{ opacity: 0, y: -8 }}, animate={{ opacity: 1, y: 0 }}, exit={{ opacity: 0, y: 8 }}`. Class includes `bg-paper text-ink font-receipt p-6 max-w-md mx-auto border border-rule receipt-print` (the receipt-print class is the CSS keyframe from Plan 01). When `stamped=true` an absolutely-positioned "PRINTED" badge appears top-right at -12deg rotation in `text-[var(--color-stamp)]`. The header renders `══` rule, header text (uppercase tracked), optional subheader, second `══` rule.

import { Modal } from '../ui/Modal'
import { Button } from '../ui/Button'
import { Receipt, ReceiptRow } from '../aesthetic/Receipt'
import { WallLabel } from '../aesthetic/WallLabel'
import type { RoundResult, GameState } from '../../types/game'
import { ARTIST_NAMES } from '../../types/game'

export function RoundEndModal({ result, game, onDismiss }: RoundEndModalProps) {
  const isGameOver = game.status === 'game_over'
  return (
    <Modal open title={`ROUND ${result.round} RESULTS`}>
      <Receipt
        header={`AUCTION RESULTS — ROUND ${result.round}`}

        stamped
      >
        <WallLabel size="sm">ARTIST RANKINGS</WallLabel>

          {result.rankings.map((r, i) => (
            <ReceiptRow
              key={r.artist}
              label={`${i + 1}. ${ARTIST_NAMES[r.artist].toUpperCase()}`}
              value={r.value > 0 ? `+$${r.value.toLocaleString()}` : '—'}
            />
          ))}
        </div>
        <div className="dotted-rule my-3" />
        <WallLabel size="sm">PAYOUTS</WallLabel>
        <div className="my-3 space-y-1">
          {result.payouts
            .map((p, idx) => ({ p, player: game.players[idx] }))
            .filter(x => x.p.amount > 0)
            .sort((a, b) => b.p.amount - a.p.amount)
            .map(({ p, player }) => (
              <ReceiptRow
                key={player?.id}
                label={(player?.displayName ?? '').toUpperCase()}
                value={`+$${p.amount.toLocaleString()}`}
              />
            ))}
        </div>
        <div className="mt-4">
          <Button variant="gold" className="w-full" onClick={onDismiss}>
            {isGameOver ? 'SEE FINAL SCORES' : `CONTINUE TO ROUND ${result.round + 1}`}
          </Button>
        </div>
      </Receipt>
    </Modal>
  )
}

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2.2-01 | Tampering | WallLabel `children` / `title` props | mitigate | React's default JSX text interpolation escapes HTML — strings flow through as text nodes, not innerHTML. No `dangerouslySetInnerHTML` is introduced. The `display name` and `title` strings ultimately come from server-validated player input (Phase 1 ENG-05 Zod schemas), so XSS via display name is already blocked at the boundary. |
| T-2.2-02 | Information Disclosure | Receipt rendering of round results | mitigate | Receipt only renders data already in the public `RoundResult` payload that the server broadcasts. No private hand or sealed bid data is read by these components. Phase 1 ENG-01/ENG-02 already enforce that bids and decks are not in the broadcast. |
| T-2.2-03 | Spoofing | Display name in winner declaration | accept | Display names are user-chosen and could be spoofed (e.g. "Server"); this is a known social-layer concern, not a security boundary. Phase 1 does not constrain display names beyond Zod length limits. v1 accepts the residual risk. |
| T-2.2-04 | Denial of Service | Receipt with very large `rows` array | mitigate | AppraisalForm and Receipt iterate over arrays whose length is bounded by game design (≤ 5 players, ≤ 5 artists, ≤ 4 rounds). No unbounded list rendering is exposed in this plan. |
| T-2.2-05 | Repudiation / Elevation | Aesthetic primitives | N/A | Pure presentation — no auth, no audit log, no privilege boundaries. |
