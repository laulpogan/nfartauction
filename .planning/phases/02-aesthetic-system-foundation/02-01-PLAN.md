<objective>
Wave 0 atomic rewrite of the visual foundation. In a single coherent set of edits, the app stops rendering on dark zinc and starts rendering on the white-base zine palette. Tailwind v4 @theme tokens are introduced for paper/ink/rule/stamp + five neighborhood accents + label/receipt/broken fonts. A NeighborhoodContext provides the active neighborhood and exposes --color-accent on its subtree. The five existing game components are stripped of every bg-zinc-* / text-white / text-zinc-* class and re-rendered against the new palette. Test infrastructure is upgraded to jsdom + @testing-library/react so subsequent plans can write component tests.

Purpose: The half-state pitfall (Pitfall 5 in RESEARCH) is the single biggest risk for this phase — if the dark theme survives anywhere, the game looks broken until phase end. This plan eliminates that risk by doing the wholesale palette swap and the context plumbing in one wave, before any primitive components or skins are built.

Output: An app that renders on off-white paper, with all existing game functionality intact, ready for primitives (Plan 02) and skins (Plan 03) to be plugged in.
</objective>

NeighborhoodContext API (src/contexts/NeighborhoodContext.tsx):
```typescript
export type Neighborhood = 'gallery' | 'warehouse' | 'flatlands' | 'hotel' | 'online'

interface NeighborhoodContextValue {
  neighborhood: Neighborhood
  accentVar: string  // e.g. 'var(--color-gallery)'
}

export function useNeighborhood(): NeighborhoodContextValue

interface NeighborhoodProviderProps {
  neighborhood: Neighborhood
  children: ReactNode
}
export function NeighborhoodProvider({ neighborhood, children }: NeighborhoodProviderProps): JSX.Element
```

The Provider MUST render a wrapping <div> with:
- `data-neighborhood={neighborhood}` attribute
- `style={{ '--color-accent': `var(--color-${neighborhood})` } as React.CSSProperties}`
- `className="neighborhood-online"` ONLY when neighborhood === 'online'

Tailwind v4 arbitrary value syntax (Pitfall 4 — VERIFY in this plan): The executor MUST run a quick test of `bg-[var(--color-accent)]`, `bg-[--color-accent]`, and `bg-(--color-accent)` against installed Tailwind 4.2.2 and pick the one that produces a CSS rule. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser DOM ↔ React | Component rendering only — no untrusted input crosses |
| Static asset CDN (Google Fonts) | External font files loaded from fonts.googleapis.com / fonts.gstatic.com |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2.1-01 | Tampering | Google Fonts CDN delivery | accept | Public CDN, no PII in request, font tampering would only change visual appearance — same risk profile as the rest of the static page assets. Subresource Integrity is impractical for Google Fonts because the served CSS rotates URLs. Documented as accepted residual risk. |
| T-2.1-02 | Information Disclosure | Google Fonts request leaks IP + referer | accept | Same risk as any third-party asset. Project does not target privacy-hostile threat models in v1; no PII transmitted with the font request. Phase 6 deployment may revisit by self-hosting fonts. |
| T-2.1-03 | Denial of Service | Google Fonts CDN unavailable | mitigate | Font fallback chain `'Special Elite', 'IBM Plex Mono', ui-monospace, monospace` ensures the app remains readable if the CDN fails. The "broken font" Online aesthetic also depends on fallback behaviour, so this is exercised by design. |
| T-2.1-04 | Elevation of Privilege | NeighborhoodProvider context | N/A | Pure UI context, no privilege boundaries. Documented as N/A. |
| T-2.1-05 | Spoofing/Repudiation | Visual rewrite of game components | N/A | No identity claims involved in the visual layer. Game engine auth (Phase 1) is unchanged. Documented as N/A. |

Note: Phase 2 is pure UI work — most STRIDE categories do not apply because no trust boundary inside the work touches authentication, authorisation, or data integrity. The threats above are the genuine ones; the rest are documented as N/A rather than padded.
