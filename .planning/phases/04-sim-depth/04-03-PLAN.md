<objective>
Add the drug inventory + Risk stat system. Drugs are acquired probabilistically from Flatlands and Hotel slots, used at parties for +Coolness/-Restedness, and tracked alongside paintings on the same AppraisalForm primitive — the appraisal of "Untitled (White), mixed media, 2024" being literally 1g of coke is the bit. Carrying more than DRUG_CONFIG.riskThreshold units accumulates passive Risk on each sim_day. Risk surfaces as a stat row on StatDisplay when > 0.

Purpose: Phase 4 differentiator #3. The drug system completes the trio of sim depth features. It also exercises the server-side entropy boundary one more time: like the dropped artist seed in 04-01, the acquisition roll lives in `party/server.ts`, not in the pure sim-engine. The aesthetic payoff is the "drugs treated like paintings on the same form" cognitive dissonance — that lives in DrugInventory using AppraisalForm.

Output: Type extension, sim-engine pure functions for inventory and risk, sim-config item data, server wiring for acquisition/use/risk, DrugInventory UI inside SimPanel, StatDisplay risk row, full test coverage.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → SUBMIT_SLOTS | Cannot inject drug items; acquisition is server-rolled |
| client → broadcast | drugs and risk must stay in PlayerSimState (private) |
| server entropy | crypto.randomUUID + Math.random for acquisition rolls — both server-only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-12 | Tampering | client-injected drug items | mitigate | No client message type adds drugs; only the server's per-slot acquisition roll calls addDrugItem; SUBMIT_SLOTS payload Zod schema does not include any drug fields |
| T-4-13 | Tampering | client-claimed party use without an item | mitigate | applyDrugEffects is gated server-side on `state.playerSim[sessionId].drugs.length > 0`; client cannot fake an item |
| T-4-14 | Information Disclosure | drugs/risk in public broadcast | mitigate | derivePublicState already strips playerSim entirely (T-3-08 carries forward) |
| T-4-15 | Tampering | risk value manipulation | mitigate | accumulateRisk is the only writer; clamped [0,100]; called only inside advanceFromSimDay |
| T-4-16 | Repudiation | acquisition rolls not logged | accept | session-scoped; logSimTransaction can surface drug events in dev mode if needed for playtest |
| T-4-17 | DoS | unbounded drugs array | mitigate | acquisition is rate-limited by slots-per-day (≤4); no client growth path; risk threshold + manual party use create natural pressure to consume |
</threat_model>
