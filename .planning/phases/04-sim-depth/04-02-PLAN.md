<objective>
Add the landlord text-message arc as a prestige-gated 5-stage progression. Each sim_day the server checks the player's current Prestige against per-stage thresholds and advances `landlordStage` (1→5) at most one step per day. Each stage has an authored message (in the zine register, 1 sentence each) that renders as an iMessage-style bubble inside SimPanel. Low prestige accelerates progression; high prestige stalls it. Stage 5 = "you're out". The arc is visible on every sim_day so the player can see it climbing.

Purpose: Phase 4 differentiator #2. The landlord arc gives Prestige (added in Phase 3 as a public mirror field) a meaningful consumer and provides the absurdist institutional-pressure beat that anchors the gallery sim's tone. It also proves the per-day server progression hook works, which 04-03 (drugs) will reuse for risk accumulation.

Output: Type extension, sim-engine pure function, sim-config message data and prestige thresholds, server wiring, LandlordMessages UI inside SimPanel, full test coverage.
</objective>

  <behavior>
    progressLandlord(playerSim: PlayerSimState, prestige: number) → { updatedPlayerSim, advanced }:
      - Reads playerSim.landlordStage (1..5) and the prestige gate from LANDLORD_CONFIG.prestigeThresholds
      - LANDLORD_CONFIG.prestigeThresholds is an array of 4 numbers — the threshold to STAY at the current stage
        - thresholds[0] = prestige needed to NOT advance from stage 1 → 2
        - thresholds[1] = prestige needed to NOT advance from stage 2 → 3
        - ...
        - Suggested values: [10, 25, 45, 70] (placeholder, low prestige loses fast)
      - If currentStage < 5 AND prestige < thresholds[currentStage - 1] → advance to currentStage + 1, push currentStage+1 onto seenLandlordStages, advanced=true
      - If currentStage = 5 → no-op (terminal stage), advanced=false
      - If prestige >= threshold → no-op, advanced=false
      - Monotonic: stage never decreases
      - Pure: no side effects
  </behavior>
