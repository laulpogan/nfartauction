<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server playerSim → GAME_OVER_APPRAISALS broadcast | private sim state is intentionally surfaced in aggregate form to all players (the appraisal IS the public end-state by design) |
| client → server (no new inbound) | this plan adds no new inbound messages — no client-side trust boundary expansion |
| reconnecting client → onConnect appraisal replay | trusted by sender.id; appraisals are public anyway |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-11 | Information Disclosure | mid-game appraisal leak | mitigate | computeFinalAppraisal is called ONLY inside the PLAY_CARD round-end branch when finalGame.status === 'game_over'; the broadcast cannot fire mid-game because the engine's status only flips at round 4 (Phase 1 ENG-08 verified) |
| T-5-12 | Information Disclosure | playerSim leakage via appraisal | accept | The appraisal IS the intentional public summary at game end. Faction mix, neighborhoods, NFT exposure, and key relationships are surfaced to all players as the end-state design (CONTEXT.md: "every player sees a printed appraisal document"). No private sim state survives into another game — sessions reset on lobby return. |
| T-5-13 | Tampering | client-supplied neighborhoodHistory | mitigate | The history is a server-only Record<string, Neighborhood[]> on ServerState, written only inside advanceFromSimDay after resolveSlots has executed. No inbound message touches it. |
| T-5-14 | Repudiation | reconnecting player misses appraisal | mitigate | lastFinalAppraisals persists on ServerState; onConnect replays GAME_OVER_APPRAISALS when game.status === 'game_over' |
| T-5-15 | DoS | computeFinalAppraisal cost | accept | O(n) over relationships (10) × O(history) (≤ 4 days) per player × ≤ 5 players = ~200 ops per round-4 transition. Negligible. |
| T-5-16 | Tampering | game-over phase forced by client | mitigate | nextPhase = { type: 'game_over' } only when finalGame.status === 'game_over', which is set by the engine's endRound function based on the cumulative round counter. No inbound message can flip this. |
</threat_model>
