<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → server CONVERT_NFT | untrusted amount + session crosses Zod validation, hits convertNft |
| client → server PURCHASE_NFT_WHITELIST | untrusted intent crosses; entropy roll happens server-side |
| server → client NFT_DM | unicast, sent only to the session that crossed the threshold |
| server → client NFT_DENOUNCEMENT | broadcast to all room connections (it's a public denouncement by design) |
| server playerSim store → public broadcast | nftWallet/heldNfts must NEVER reach derivePublicState |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 | Tampering | CONVERT_NFT amount | mitigate | Zod `.int().min(1).max(1000)`; convertNft additionally rejects amount > nftWallet inside the pure function |
| T-5-02 | Tampering | PURCHASE_NFT_WHITELIST cost bypass | mitigate | NFT_CONFIG.whitelistCost is a server constant; client sends no amount; handler checks `playerSim.nftWallet >= NFT_CONFIG.whitelistCost` before calling the engine |
| T-5-03 | Spoofing | CONVERT_NFT from unknown session | mitigate | sender.id → state.sessions lookup; reject if not present, mirroring SUBMIT_SLOTS handler |
| T-5-04 | Information Disclosure | nftWallet / heldNfts in public broadcast | mitigate | Both fields live on PlayerSimState; derivePublicState already strips playerSim entirely (Phase 3 T-3-08 invariant). Verified: 0 references to playerSim or nftWallet in derivePublicState body. |
| T-5-05 | Tampering | NFT unlock bypass | mitigate | nftWalletUnlocked is set ONLY inside advanceFromSimDay's threshold-cross detector; no inbound message can flip it; CONVERT_NFT handler rejects when !nftWalletUnlocked |
| T-5-06 | Tampering | Faction reaction bypass | mitigate | The relationship updates happen in the PURCHASE_NFT_WHITELIST handler unconditionally (every successful purchase triggers them); the client cannot omit them |
| T-5-07 | Information Disclosure | NFT_DM leaked to wrong session | mitigate | Server iterates room.getConnections() and only sends NFT_DM to the connection whose conn.id === sessionId of the player that crossed |
| T-5-08 | DoS | Spam CONVERT_NFT | accept | Zod cap on amount; PartyKit message rate-limit handles flooding upstream; no per-room state grows unbounded |
| T-5-09 | Tampering | hype drift manipulation | mitigate | Math.random for drift lives in party/server.ts only; client has no input that touches sim.nftHypeCycle directly; applyNftHypeDrift clamps to [0,100] |
| T-5-10 | Repudiation | NFT activity audit log | accept | Session-scoped game; logSimTransaction in dev mode is sufficient for playtesting |
